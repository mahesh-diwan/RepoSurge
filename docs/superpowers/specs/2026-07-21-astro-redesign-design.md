# RepoSurge Astro Redesign

**Date:** 2026-07-21
**Status:** Approved
**Stack:** Astro 5.x, TypeScript, Tailwind CSS, vanilla JS

## Goal

Migrate RepoSurge from Next.js to Astro and expand features while keeping the brutalist aesthetic polished.

## Approach

Pure Astro — zero client JS except where needed (search, sort, preferences). No React, no Alpine, no frameworks.

## Architecture

```
src/
├── content/
│   └── repos.json          # repo data (generated by fetch script)
├── layouts/
│   └── Base.astro          # HTML shell, nav, footer
├── pages/
│   ├── index.astro         # Home — rankings list
│   ├── repo/[slug].astro   # Repo detail page
│   └── about.astro         # About/methodology
├── components/
│   ├── Header.astro        # Hero section
│   ├── RepoCard.astro      # Single repo row (static)
│   ├── StarChart.astro     # CSS bar sparkline
│   ├── SearchBar.astro     # Client-side search (<script>)
│   ├── FilterBar.astro     # Time tabs + language filter
│   ├── SortSelect.astro    # Sort dropdown
│   └── TrustLogos.astro    # Logo strip
├── styles/
│   └── global.css          # Design tokens + base styles
└── scripts/
    └── fetch-repos.ts      # Data fetching (build time)
```

**Routing:** File-based. `/` = home, `/repo/[slug]` = detail, `/about` = about.

**Data flow:** `fetch-repos.ts` runs at build → writes `content/repos.json` → Astro pages read from it at build time → SSG.

## Design System

### Tokens

| Token         | Value            | Usage                  |
| ------------- | ---------------- | ---------------------- |
| `--midnight`  | `#0A0A0A`        | Backgrounds            |
| `--bone`      | `#F5F5F0`        | Text, borders          |
| `--electric`  | `#0066FF`        | Accents, active states |
| `--red`       | `#FF3333`        | Negative trends        |
| `--font-mono` | `JetBrains Mono` | Everything             |

### Typography

- Hero: `text-5xl` (48px) — title only
- Section heads: `text-lg` (18px) — uppercase, tracking-widest
- Body: `text-sm` (14px) — repo descriptions, metadata
- Small: `text-xs` (12px) — labels, tabs

### Polish (brutalist DNA preserved)

- 1px borders everywhere
- `hover:bg-bone/5` subtle hover states
- `transition-colors` on interactive elements (150ms)
- `active:scale-[0.98]` press feedback on buttons
- Focus rings: `outline-electric` for a11y
- Zero border-radius, monospace everywhere, high contrast, no gradients, no shadows

## Pages

### Home (`/`)

- Hero: "REPØSURGE" title + tagline + CTA → scrolls to rankings
- TrustLogos strip
- FilterBar: time tabs + language pills
- SearchBar: text input for filtering repos
- SortSelect: dropdown for sort order
- RepoCard list: ranked rows with star chart + velocity
- Footer

### Repo Detail (`/repo/[slug]`)

- Back link
- Repo name (large), description, language, stars
- Full-width star history chart
- Stats grid: stars, gained, velocity, language
- Link to GitHub

### About (`/about`)

- What RepoSurge is
- How velocity is calculated
- Data sources and refresh rate

## Interactivity

**Search:** Client-side `<input>` with `<script>` that filters visible repo cards by name/description match.

**Sort:** `<select>` dropdown — options: "Stars gained" (default), "Total stars", "Velocity", "Name". Client-side reorder of DOM elements.

**Filters:** Time tabs + language pills are `<a>` links that reload the page with query params. SSG handles data filtering at build time.

**Local preferences:** `localStorage` stores `preferred_language` and `preferred_sort`. Loaded on page init, applied to DOM.

**Repo detail:** Static page, no interactivity beyond navigation links.

**No client-side routing** — full page loads.

## Data

- `fetch-repos.ts` script fetches from GitHub API (same as current `lib/github.ts`)
- Writes to `src/content/repos.json`
- Per-repo slug derived from `full_name` (e.g., `vercel/next.js` → `vercel-next-js`)
- Build generates static pages for each repo

## Key Decisions

1. **No React** — Astro components + vanilla JS for interactivity
2. **SSG only** — all data at build time, no server runtime
3. **localStorage for preferences** — no backend needed
4. **File-based routing** — standard Astro pattern
5. **Monospace everywhere** — brutalist identity preserved
