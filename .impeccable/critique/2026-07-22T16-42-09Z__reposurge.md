## Overall Impression

RepoSurge is a genuinely well-executed terminal-brutalist tool. The aesthetic is committed and disarming — it looks like something a solo developer built for themselves, which is the highest compliment for a developer tool. The live polling with green delta badges is the standout feature: instantly legible, emotionally engaging, technically graceful.

The single biggest opportunity is fixing the `#666` contrast — it's the one thing holding back an otherwise excellent execution. Bump to `#999` and the accessibility story flips from "blocker" to "clean."

## What's Working

1. **Cohesive identity.** Terminal-green-on-midnight is committed, not decorative — `>` prompt in search, JetBrains Mono, flat rows, every detail reinforces the metaphor.
2. **Live polling UX.** Green dot with glow + live label + per-repo deltas is instantly legible. Delta timing as superscript text is unobtrusive — no modals, no toasts.
3. **Layout restraint.** No cards, no shadows, no gradients. Flat rows maximize scanability. Mobile preserves the same visual grammar.

## Priority Issues

### P1: Low contrast on ALL dim text (#666 on #0A0A0A = 3.4:1)

- **What:** `text-dim` = `#666` on `#0A0A0A` = 3.4:1, fails WCAG AA 4.5:1. ≈15 instances across nav links, labels, rank numbers, footer, about body.
- **Why:** Hard to read on low-quality displays / bright light. WCAG failure.
- **Fix:** Bump `dim` to `#999` or `#888` (≈5:1 on `#0A0A0A`).
- **Suggested:** `/impeccable colorize`

### P1: No heading hierarchy (About page)

- **What:** About page has zero `<h1>`–`<h3>`. All `<p>` tags. Screen readers can't navigate.
- **Why:** WCAG failure. SEO degraded.
- **Fix:** Wrap "what is this", "how it works", "stack" in `<h2>`.
- **Suggested:** `/impeccable audit`

### P2: Repo name text-overflow on mobile (17–101px overflow)

- **What:** `truncate` class doesn't contain overflow. Long names push `gained` values off-screen.
- **Why:** Layout breaks on narrower viewports.
- **Fix:** Add `max-w-[160px]` or equivalent constraint.
- **Suggested:** `/impeccable adapt`

### P2: No active nav state on desktop

- **What:** All nav links show `text-dim` regardless of current page. No `aria-current="page"`.
- **Why:** First-timers lose orientation. Power users find it inefficient.
- **Fix:** Highlight matching link with `text-terminal` + `aria-current`.
- **Suggested:** `/impeccable harden`

### P3: Empty search lacks guidance

- **What:** "no repos match vercel" — functional but no suggestions or next steps.
- **Why:** Stress testers hit a dead-end. First-timers don't know what to do.
- **Fix:** Show "search by owner name" hint or popular repo names below empty state.
- **Suggested:** `/impeccable clarify`

## Persona Red Flags

**Alex (Power User):** No keyboard shortcuts. No `s` for search, `/` for filter, `1-4` for periods. Sort toggle is subtle (triple-click to clear). No URL-persisted sort/filter — state lost on navigation. **Risk: Moderate.**

**Casey (Mobile User):** Search + period picker at top of 375px viewport — hard to reach one-handed. Text-overflow on long names breaks the layout. No scroll-position preservation on tab switch. **Risk: Low.**

**Sam (Accessibility-Dependent):** WCAG AA failures on contrast (3.4:1 vs 4.5:1) — P1. No heading hierarchy — screen readers hit flat text on About page. `:focus-visible` styled correctly with green outline + offset. All interactive elements are `<a>`/`<button>`. ARIA labels present. **Risk: High.**

## Questions to Consider

1. What if repo rows were clickable anywhere instead of only the name link? The rank/gained areas do nothing on click — trivial ergonomic gain.
2. What if search accepted GitHub-specific syntax? `owner:vercel`, `lang:rust` — the terminal aesthetic is a natural fit for a query-language affordance.
3. What if period navigation used swipe gestures on mobile? Horizontal swipe between daily/weekly/monthly would reduce the top-of-screen thumb zone issue.
